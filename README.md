# Gemini 的簡單歷史課 🏛️

這是一個專為「歷史專題研究與報告」打造的自動化網頁生成專案。結合了 Markdown 的便利性與現代化網頁設計，讓長篇歷史文章能以最優雅、易讀的方式呈現。

👉 **[目前上線的網站](https://ludwicia.github.io/gemini-history-lesson/)**

---

## 📝 專案工作紀錄 (Work Log)

本專案由 USER 與 Antigravity (Gemini) 共同協作完成，核心成就包含：

1. **自動化轉換架構建立**
   - 開發 `build_html_md.py` Python 腳本，將純文字 Markdown 自動轉譯為具備完整結構的 HTML5 網頁。
   - 撰寫預處理邏輯，完美解決 Word 格式轉 Markdown 時常見的單行換行（文字牆）問題，確保段落分明。
2. **版面視覺設計與互動開發**
   - **動態首圖橫幅**：設計結合 AI 與歷史感氛圍的高質感 Banner，配有漸層文字與淡入動畫。
   - **三欄式響應式佈局**：左側固定為自動生成的章節目錄 (TOC)；中間為易讀的內文區塊；右側為實用工具與版權宣告。
   - **關鍵字即時搜尋**：透過原生 JavaScript 開發前端搜尋引擎，可即時高亮關鍵字並自動捲動定位至內文段落。
3. **圖文視覺整合**
   - 自動從 Wikimedia Commons 引入具備歷史價值的高畫質開源圖片（如 1658 聯省共和國地圖、熱蘭遮城等）。
   - 實作「圖片縮小至 1/4 並靠左浮動」的文繞圖 (Text Wrap) 效果，提升歷史長文的閱讀體驗。
4. **全自動雲端部署**
   - 將專案資料夾初始化為 Git 儲存庫，並自動推送到 GitHub。
   - 透過 GitHub CLI 開啟 GitHub Pages，達成「一鍵推播即可更新網站」的極致流暢工作流。

## 📌 版本控制規範 (Versioning Rules)

本專案採用**雙軌獨立版本號**設計，以清晰區分「網頁排版設計」與「歷史文章內容」的異動：

### 1. 🎨 版面設計版本 (Layout & Design Version)
* **用途**：追蹤網頁的外觀設計、CSS 排版、新增互動功能（如摺疊、搜尋功能）以及底層腳本 (`build_html_md.py`) 的異動。
* **顯示位置**：網頁右下角（或手機版懸浮底欄）的「📝 版本宣告」卡片。
* **命名規則**：
  * 當發生**重大視覺改版**或結構性功能重構時，提升主版號（例：`v1.0` ➡️ `v2.0`）。
  * 當發生**局部功能優化**、小排版微調或 bug 修復時，提升副版號（例：`v1.0` ➡️ `v1.1`）。
* **目前版本**：`3.4` *(已支持多歷史專題與多三欄歷史文件自適應排版、全域搜尋與雙路由模組)*

### 2. 📖 內容版本 (Content Version)
* **用途**：追蹤特定歷史文章的內容修訂、史實勘誤、錯字修正或段落補充。
* **顯示位置**：文章大標題 (`<h1>`) 正下方顯眼的藍色小標籤。
* **命名規則**：
  * 這是**針對各篇文章獨立計算**的版本號。
  * 初次發布文章時，內容版本為 `1.0`。
  * 若後續修正文字、補充史實或更新內容，則提升該文章的內容版本（例：`1.0` ➡️ `1.1`）。
* **目前版本**：`1.0` *(荷蘭史初始版本)*

## 🚀 下一堂歷史課：使用教學

當你要發布新的歷史文章時，請遵循以下步驟：

### 第一步：準備 Markdown 文件
1. 將你的新歷史課程內容存成 `.md` 檔案（例如：`第二堂課：法國大革命.md`），並放進這個專案資料夾。
2. 確保文章內容有使用適當的標題層級（如 `## 1. 標題`），這會用來自動生成左側目錄。

### 第二步：修改轉換腳本
打開 `build_html_md.py`，進行以下微調：
1. **修改讀取檔名**：將腳本最上方的 `with open(...)` 檔名改為你的新 Markdown 檔名。
2. **更新圖片字典 (Optional)**：如果新文章需要專屬的維基百科圖片，可以修改腳本裡的 `images` 字典，將章節編號對應到新的圖片 URL 與圖說。

### 第三步：生成網頁
打開終端機，確保路徑在專案資料夾下，執行：
```bash
python build_html_md.py
```
這時資料夾內的 `index.html` 就會被覆寫為最新內容的網頁！你可以直接點開 `index.html` 檢查排版與圖片。

### 第四步：推播上線
確認本機端的網頁沒問題後，在終端機輸入以下三個指令，將新內容推送到全球網路：
```bash
git add .
git commit -m "發布新課程：法國大革命"
git push
```
等待約 1~2 分鐘後，重新整理你的 [GitHub Pages 網址](https://ludwicia.github.io/gemini-history-lesson/)，新的歷史課就正式開課囉！
